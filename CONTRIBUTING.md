# Contributing to IBRASERVICES

Thank you for your interest in contributing to IBRASERVICES! This document provides guidelines and instructions for contributing.

## Code of Conduct

Be respectful and inclusive in all interactions.

## How to Contribute

### Reporting Bugs

1. Check existing issues to avoid duplicates
2. Create a new issue with:
   - Clear title
   - Description of the problem
   - Steps to reproduce
   - Expected vs actual behavior
   - Screenshots if applicable

### Suggesting Enhancements

1. Use GitHub Issues to suggest features
2. Provide clear description and use cases
3. Include mockups or examples if helpful

### Pull Requests

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Follow code style guidelines
4. Commit with clear messages
5. Push to your fork
6. Create a Pull Request with:
   - Clear description
   - Reference to related issues
   - Screenshots for UI changes

## Development Setup

```bash
# Clone and setup
git clone https://github.com/ibrahima02ndiaye-debug/IBRASERVICES.git
cd IBRASERVICES
npm run install-all

# Start development
npm run dev
```

## Code Style

- Use TypeScript for type safety
- Follow existing code patterns
- Run linting: `npm run lint`

## Testing

- Write tests for new features
- Run tests before submitting PR: `npm test`

## Commit Messages

Use clear, descriptive commit messages:
- `feat: Add new feature`
- `fix: Fix bug in component`
- `docs: Update documentation`
- `style: Code formatting`
- `refactor: Restructure code`

Thank you for contributing!
